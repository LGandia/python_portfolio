TOP THREE CHALLENGES IN INPLEMENTATION

  1. Using and creating Flexboxes

    Previously having very little experience with CSS and HTML it was a challenge to tackle the implementation and use of flexboxes, and I am also not very good with object oriented programing outside of the basics used in Pygame which was used in my A-level project. This created a big challenge as I was unsure how to set up the container and each individual box.

  2. Placing and editing the footer

    In the resume, due to the invisible flexboxes, it was hard to find the right place to put the footer, as due to the different types of flex orders and wrappers I used, the footer would always shift to a different row or column than I intended. Another challenge with the footer was setting up a button that worked, and placing down the icons; all in the right place and alignment. I also found it particularly challenging to find a way to style the icons, and make both interactive with the lack of JavaScript

  3. Adjusting the header to overlap with the image

    The flexboxes stop the header from being able to overlap with other elements, and taking the header out of the flex container shifts the position of everything, similarly to the footer. However, unlike the footer, by taking out the header from the container, it would cause it to size up significantly and also create a new row/column and still not overlap with the elements I wished it did. This also caused the flexboxes under the heading to shift up and the width of the boxes to decrease, making the text in education to overlap with each other.

HOW I MANAGED THE CHALLENGES

  1. Flexboxes

    Through a lot of research and analysing other people's codes, I was able to figure out how flexboxes work and how to set them up so that they function how I want them to. After practicing a lot in a separate document with other information I had gathered from textbooks and guides, I started to become more confident with the flexboxes and started to code the website. As mentioned above it was hard to place some elements on the website that didn't quite align with the flexboxes, but through a lot of trial and error I did manage to place everything where I believe it should be.

  2. Placing and editing the footer

    Through a lot of trial and error, and making small adjustments to figure out what affected the size and placement of the footer, I was able find the right settings and place for the footer. Another challenge along with this was the styling and interactional (hover features) of the icon. But after some trial and error, as well as doing some research I managed to find the right set of properties that needed to be given for the icons to be the right colour, as well as change to be a different colour when hovered upon. This was a brand new thing for me to learn. Additionally, I found out that the "button" tag could not have a link anchored to it, so I had to re-edit my code after validating it to create a brand new button without it being an actual button, as to make it functional.

  3. Adjusting the header to overlap with the image

    After a long time of researching I have managed to find the solution to the problem, by making the header's position absolute. However, this caused another wave of problems as mentioned above, since then the header was fixed at the top, and the rest of the flexboxes would push up and shrink, with the contacts section being completely covered by the overlapping heading. This was solved my first placing the header where I wanted it to be, by adding the right padding at the top, and then adding a top padding to the contacts section which allowed it to shuffle everything down. As for the shrinking, I opted to set the flex shrink to 0, which stops the flexbox from shrinking, rather than attempt to make it bigger (using grow) since it was not working previously.
